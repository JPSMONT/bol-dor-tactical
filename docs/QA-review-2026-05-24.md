## Open issues (prioritised)

| # | Area | Severity | Issue |
|---|---|---|---|
| 1 | Offline / service worker | **High (race-day)** | `sw.js` runtime-caches **only** Open-Meteo. MeteoSwiss station data (`data.geo.admin.ch`) is never written to cache → unavailable offline. Leaflet + JSZip (from unpkg) are not precached in `ASSETS` → the map / replay can fail on an offline cold-start. `CLAUDE.md` line 48 overstates this ("caches … MeteoSwiss responses"). Matters in the no-signal Haut-Lac stretch. |
| 2 | GPS battery | Medium | `watchPosition` uses `{enableHighAccuracy:true, maximumAge:1000}` — continuous high-accuracy fixes for an 18–24h race is the heaviest possible battery draw. |
| 3 | Race-time constants | Medium (fragility) | Start time is hardcoded in **three** places (lines 923, 1863, 3191 `BOL_DOR_START`) plus three different end constants (`RACE_END_MS` T+20h, `BOL_DOR_FINISH` T+24h, `BOL_DOR_END` T+36h). All currently agree, so no live bug — but a change must touch 3 copies. The handoff's "verify countdown" note only lists 923 + 1863 and misses 3191. |
| 4 | Live BOL26 polling | Info (deferred / blocked) | `toggleLiveMode()` runs a 60-min timer but `poll()` just reloads the selected historical year as a stand-in. Real 2026 live positions are blocked until SuiviRegate publishes the `BOL26` race code. |
| 5 | Doc drift — `CLAUDE.md` | Low | Line 5 says the course "finishes at Le Bouveret"; the app correctly models a round trip finishing at Genève (7 waypoints). The `WAYPOINTS` snippet (lines 108–114) shows 5 one-way points vs the real 7-point route. |
| 6 | Doc drift — `README.md` | Low | Line 38 says countdown to "09:00 CEST"; code, handoff, and official program all say **10:00**. |

### Recommendations

**Issue 1 (do before T5).** In `sw.js`, extend the network-first-then-cache branch to also match `data.geo.admin.ch`, and add the pinned unpkg Leaflet + JSZip URLs to the `ASSETS` precache list (or self-host them alongside the app). Bump cache `v4 → v5`. Then test in airplane mode on the dry run. This is the single highest-value reliability fix before race day. *Assumption: the unpkg URLs in `index.html` are version-pinned; if they're floating (`@latest`), pin them first so precache hashes are stable.*

**Issue 2.** Add a low-power overnight GPS mode (drop `enableHighAccuracy`, raise `maximumAge`, reduce cadence) toggled manually or after sunset, and measure the battery curve over 2h+ at T5. Keep the two-phone + power-bank plan as the backstop.

**Issue 3.** Define the start time once and derive `RACE_END_MS` / `BOL_DOR_FINISH` / `BOL_DOR_END` from it. Low risk, prevents a future mismatch.

**Issue 4.** No action until the `BOL26` code is published — check on registration day (4 June), then wire the real feed + live map display.

**Issues 5–6.** One-line doc fixes, pending your go-ahead (they were outside the handoff §6 scope I was given).
